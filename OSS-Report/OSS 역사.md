# 1950~1960년대


# 1960~1970년대
1969년
- 아파넷 등장
  - 미국 국방부 연구기관인 (ARPA)의 주도하에 만들어진 세계 최초의 패킷 스위칭 네트워크.
  - 현재 인터넷의 원형이며 1969년에 시작되어 1970년에 걸쳐 확장.
  - 초기의 소프트웨어 및 코드 공유, 네트워크 역할.
  - 오픈소스 문화가 네트워크를 통해 확산되는 기반 역할.


# 1970~1980년대
1970년대 UNIX 개발

1969~1970년
- 벨 연구소에서 Unix 개발, 1970년대에 처음 배포.
   - AT&T를 비롯한 여러 회사, 버클리 등 비영리 단체에서 다양한 버전 개발.

# 1980~1990년대
1980년대 : 소프트웨어의 상업화
많은 기업들이 소스 코드를 폐쇄하고 소프트웨어를 독점적으로 판매하기 시작.
이러한 변화는 개발자들 사이에서 자유로운 소프트웨어 사용과 공유를 제안하는 움직임으로 받아들여짐.

1983년
-자유 소프트웨어 운동 (Free Software Movememt)
 - 리처드 스톨만이 소프트웨어의 본래 생산 유통 방식인 정보 공유 방식을 복원하고자 한 운동.
  - GNU 프로젝트 기반, 목표는 운영체제만이 아닌 모든 소프트웨어를 자유 소프트웨어로 만드는 것.

1985년 
- 자유 소프트웨어 재단 설립 (FSF)
  - GNU 선언문 재정과 카피레프트 운동 주창



# 1990~2000년대
1990년대 초반 (1990~1994): 기반 형성기
1990년
- 월드 와이드 웹(www)의 탄생
   - 팀 버너스 리가 HTTP, HTML, 웹 브라우저를 개발하여 인터넷 접근 방식에 혁신을 가져옴.
   - 오픈 표준의 중요성을 인식하게 된 계기.

1991년
- 리눅스의 탄생,핀란드 헬싱키 대학의 대학원생인, 리누스 토발즈가 취미로 개발한 커널,핀란드 대학 3학년,MINIX를 응용하여 리눅스 커널 개발 후 GUN GPL로 배포

1992년 : 리눅스 커널과 GUN의 통합
-  GNU 프로젝트의 도구들과 리눅스 커널이 결합하여 UNIX 대안으로 발전.
- 오픈소스 생태계의 초석이 마련됨.
  
1993년 
- 리눅스 배포판의 등장
   - Slackware: 가장 초기의 리눅스 배포판으로, 간단하면서도 강력한 시스템.
   - Debian: 자유 소프트웨어 원칙을 기반으로 한 배포판으로, 이후 강력한 커뮤니티 형성.
   - FreeBSD: BSD 계열의 강력한 운영체제로 출시.

1994년 
- 리눅스 커널 1.0 버전 발표
   - 안정화된 리눅스 커널의 첫 공식 릴리스.
   - 리눅스가 학술·개인 프로젝트에서 상업적, 공공적 활용으로 확장되는 계기.

1990년대 중반 (1995~1997): 폭발적 성장기
1995년
- 아파치 웹 서버 출시
   - NCSA HTTPd에서 파생된 Apache HTTP Server가 출시.
   - 뛰어난 성능과 오픈소스 라이선스를 통해 웹 서버 시장의 표준으로 자리 잡음.
- PHP와 MySQL의 등장
   - PHP: 동적 웹 페이지 생성을 위한 서버 사이드 스크립트 언어.
   - MySQL: 빠르고 가벼운 관계형 데이터베이스로 웹 애플리케이션 개발의 핵심 도구로 성장.
- Red Hat Linux 출시
   - Red Hat은 리눅스를 상업적으로 패키징하여 엔터프라이즈 시장에 진출.
   - 오픈소스 상업화의 성공 사례로 자리 잡음.

1996년
- KDE 프로젝트 시작
   - 리눅스의 사용자 친화적인 데스크톱 환경을 제공하기 위해 시작.
   - Qt 라이브러리를 기반으로 개발.
- GIMP 출시
   - GNU Image Manipulation Program: 오픈소스 이미지 편집 도구로, 상업적 소프트웨어의 대안으로 자리 잡음.
  
1997년
- GNOME 프로젝트 시작
   - KDE가 독점적 라이브러리(Qt)를 사용하는 것에 대한 대안으로 GNU 라이선스를 사용하는 데스크톱 환경을 개발.
- IBM의 리눅스 도입
   - 대형 IT 기업이 리눅스의 가능성을 인식하고 지원하기 시작.

1990년대 후반 (1998~1999): 전환기
1998년
- 넷스케이프 브라우저 소스코드 공개
   -넷스케이프는 브라우저 시장에서 마이크로소프트와의 경쟁 심화 속에서 소스코드를 공개하고 Mozilla 프로젝트를 시작.
   -이 사건은 "오픈소스"라는 용어를 대중화하는 계기가 됨.
- 오픈소스 이니셔티브(OSI) 설립
   - 브루스 페렌스와 에릭 레이몬드가 OSI를 설립하여 오픈소스 정의와 인증 절차를 개발.
   - 오픈소스를 단순히 무료 소프트웨어가 아닌 협력과 품질 보장의 모델로 정의.
- 리눅스의 상업적 성공
   - Dell, IBM, HP와 같은 대형 IT 기업이 리눅스를 도입하며, 신뢰성과 안정성이 입증됨.

1999년
- Red Hat 상장 (NASDAQ)<br>
   - 레드햇이 NASDAQ에 상장하며, 오픈소스 소프트웨어의 상업적 가능성을 증명.
- 데스크톱 리눅스의 도전
   - KDE와 GNOME의 성숙으로 리눅스가 데스크톱 운영체제로 도전장을 내밂.
- CVS와 버전 관리 도구의 보편화
   - 협업 중심의 개발 방식에서 버전 관리 시스템(CVS)이 중요한 역할을 함.

  <h3>1990년대 주요 특징</h3>
  
1. 리눅스와 GNU 프로젝트의 결합
- UNIX 대안을 제공하며, 상업용 시스템에 대한 대안으로 자리 잡음.

2. 아파치와 웹 기술의 확산
- 오픈소스 웹 서버와 도구가 인터넷 성장의 핵심 역할을 수행.

3. 상업적 성공과 기업 참여
- 레드햇, IBM, HP 등 대기업의 지원으로 오픈소스의 신뢰성과 비즈니스 모델이 강화.

4. 커뮤니티와 협업 문화의 정착
- 인터넷과 메일링 리스트를 통한 글로벌 개발자 협업이 활발히 이루어짐.

5. 오픈소스 철학과 정의의 확립
- OSI와 GPL의 등장으로 오픈소스의 철학과 법적 기반이 확고해짐.
  
# 2000~2010년대
2000년대 초반 (2000~2004):안정화와 확산
2000년 
- IBM의 리눅스 지원 선언
   - IBM은 10억 달러를 투자해 리눅스를 지원한다고 발표
   - 리눅스와 오픈소스 기술이 엔터프라이즈 환경에서 신뢰받는 선택지가 됨.
 
2001년
- 오픈소스 라이선스 논란과 발전
   - GPL 2.0의 영향이 커지면서, 오픈소스와 상업 소프트웨어 간의 법적 논쟁이 본격화.
   - 기업들이 오픈소스 소프트웨어를 사용하면서도 라이선스 의무를 인지하기 시작.
- Wikipedia의 설립 (2001년 1월)
   - 오픈소스 철학을 콘텐츠로 확장한 사례.
   - 협업과 공유를 통해 누구나 기여할 수 있는 대중적 지식 플랫폼.
- Mono 프로젝트 시작
   - 마이크로소프트의 .NET 기술을 오픈소스로 구현하기 위한 프로젝트.
2002년
- Mozilla Foundation 설립
   - Netscape의 소스코드 공개에서 시작된 Mozilla 프로젝트가 공식 재단으로 전환.
   - 이후 Firefox 브라우저(2004년)가 출시되어 오픈소스 브라우저 시장을 주도.
- OpenOffice.org 출시
   - 썬 마이크로시스템즈(Sun Microsystems)가 StarOffice를 기반으로 공개.
   - 마이크로소프트 오피스의 대안으로 자리 잡음.

2003년
- SUSE Linux Enterprise 출시
   - SUSE는 기업용 리눅스 배포판을 출시하며 RHEL과 경쟁.
- MySQL의 확산
   - LAMP 스택(Linux, Apache, MySQL, PHP/Perl/Python)이 웹 개발의 표준으로 자리 잡음.
- Subversion 발표
   - 오픈소스 버전 관리 도구로, 기존 CVS의 문제점을 개선.
   - 
2004년
- Ubuntu 리눅스 첫 출시 (2004년 10월)
   - 데비안 기반으로 사용자 친화성을 강조한 배포판.
   - "리눅스 데스크톱의 대중화"를 목표로 빠르게 확산.
- Firefox 브라우저 출시 (2004년 11월)
   - 첫 주에 100만 회 이상 다운로드되며 IE 독점을 타파하는 데 성공.

2000년대 중반 (2005~2007): 도구와 생태계 확장

2005년
- Git의 개발
   - 리누스 토르발스가 분산형 버전 관리 시스템인 Git을 개발.
   - 오픈소스 프로젝트 관리의 혁신적 도구로 자리 잡음.
- Creative Commons 라이선스 확산
   - 콘텐츠 공유를 촉진하기 위한 라이선스 모델이 오픈소스 문화와 결합.
2006년
- Amazon Web Services(AWS)의 등장
   - 오픈소스 소프트웨어와 클라우드 기술이 융합.
   - LAMP 스택과 같은 오픈소스 도구가 클라우드 환경에서 폭넓게 사용되기 시작.
- Red Hat, 1조 원 매출 달성
   - 오픈소스의 상업적 성공을 증명하며, 기업 시장에서 신뢰를 얻음.
2007년
- Android 공개
   - 구글이 리눅스 기반 모바일 OS인 안드로이드를 공개.
   - 오픈소스와 상업적 생태계의 조화를 보여준 사례.
- Hadoop 1.0 출시
   - 오픈소스 기반의 빅데이터 플랫폼으로, 대규모 데이터 처리 기술을 선도.
   - 
2000년대 후반 (2008~2009): 현대 오픈소스의 기틀

2008년
- Chrome 브라우저 출시
   - 구글이 Chromium 프로젝트를 기반으로 Chrome 브라우저를 출시.
   - 빠르고 가벼운 브라우저로 Firefox와 경쟁.
- Android 첫 상용화
   - HTC Dream(또는 G1) 스마트폰이 안드로이드를 탑재하여 출시.
   - 모바일 오픈소스 생태계 확장의 시작.
- OpenStreetMap의 주목
   - 사용자 협력을 통해 생성된 오픈소스 지도 서비스로, 상업용 지도 서비스에 대안 제공.

2009년
- GitHub 설립
   - Git 기반의 협업 플랫폼으로, 오픈소스 프로젝트 관리와 개발자 협업의 중심지가 됨.
- Oracle의 Sun Microsystems 인수
   - Java와 MySQL을 포함한 주요 오픈소스 자산을 획득하며, 오픈소스 커뮤니티에 영향을 미침.
- MongoDB 출시
   - NoSQL 데이터베이스로, 빅데이터와 클라우드 환경에서 사용되며 급성장.

  <h3>2000년대 주요 특징 요약</h3>

1. 오픈소스와 상업 소프트웨어의 융합
- IBM, Red Hat, SUSE 등의 기업이 오픈소스 기술로 수익을 창출.
- 오픈소스 기반 상업용 서비스 모델이 정착.

2. 웹과 클라우드의 확산
- Apache, MySQL, PHP로 구성된 LAMP 스택이 웹 개발의 주류가 됨.
- 클라우드 환경에서 오픈소스 기술의 활용 증가.

3. 도구와 협업 문화의 강화
- Git과 GitHub의 등장으로 협업 방식이 혁신.
- Subversion, Creative Commons 등 협업 도구와 라이선스 모델 확산.

4. 사용자 중심의 프로젝트 증가
- Ubuntu, Firefox, Android 등 사용자 친화적인 오픈소스 소프트웨어가 폭발적 성장을 이룸.

5.대기업의 적극적인 참여
- IBM, 구글, Oracle 등이 오픈소스 프로젝트를 지원하거나 통합하여 생태계를 확장.
  
# 2010~현재
2010년대 초반 (2010~2014): 컨테이너화와 클라우드 중심의 성장
2010년
- OpenStack 프로젝트 시작
   - NASA와 Rackspace가 협력해 클라우드 컴퓨팅 플랫폼인 OpenStack을 출시.
   - 오픈소스 클라우드 생태계의 주요 프로젝트로 성장.
- Oracle, OpenSolaris 종료
   - Oracle이 OpenSolaris 프로젝트를 종료하면서 오픈소스 커뮤니티에서 논란.

2011년
- Hadoop의 빠른 성장
   - 빅데이터 분석 플랫폼으로 Hadoop이 주목받으며, 데이터 중심 애플리케이션의 기반으로 자리 잡음.
- Docker 프로젝트 시작
   - 컨테이너 기반 애플리케이션 배포를 혁신한 Docker가 출시.
   - DevOps 문화와 컨테이너 기술의 급격한 확산을 이끌어냄.

2012년
- OpenStack Folsom 릴리스
   - OpenStack의 기능이 성숙하며 클라우드 인프라 솔루션으로 자리 잡음.
- Ansible 프로젝트 출시
   - 간단하고 강력한 IT 자동화 도구로, DevOps 환경에서 널리 사용됨.

2013년
- Kubernetes의 초기 개발 시작
   - 구글 내부에서 시작된 Kubernetes 프로젝트는 컨테이너 오케스트레이션 기술의 기반을 다지기 시작.
   - 이후 클라우드 네이티브 애플리케이션의 표준으로 자리 잡음.
- Edward Snowden 폭로
   - 정부 감시에 대한 폭로 이후, 오픈소스 보안 소프트웨어에 대한 관심 증가.

2014년
- Kubernetes 프로젝트 공개 (구글)
   - 컨테이너 오케스트레이션 도구로, DevOps와 클라우드 네이티브 기술의 핵심 도구로 자리 잡음.
   - CNCF(Cloud Native Computing Foundation)의 주요 프로젝트.
- Let’s Encrypt 프로젝트 시작
   - HTTPS 보안을 무료로 제공하는 프로젝트로, 웹 보안의 표준화를 가속화.

2010년대 중반 (2015~2017): 클라우드 네이티브와 AI의 확산

2015년
- TensorFlow 출시 (구글)
   - AI/머신러닝을 위한 오픈소스 프레임워크로, 데이터 과학 및 AI 생태계의 핵심 도구로 자리 잡음.
- Hyperledger 프로젝트 시작
   - 리눅스 재단 주도로 블록체인 기술의 상업적 활용을 위한 오픈소스 프로젝트로 시작.
- Atom 에디터 출시
   - GitHub이 개발한 오픈소스 코드 편집기로 개발자 커뮤니티에서 큰 인기를 끔.

2016년
- Microsoft, SQL Server 리눅스 지원 발표
   - 오픈소스 생태계와 협력을 강화하며 리눅스 지원을 선언.
- VS Code 공개
   - Microsoft가 Visual Studio Code를 오픈소스로 공개하며 개발 도구 시장에서 선두로 자리 잡음.
- Apache Spark의 인기 상승
   - 대규모 데이터 처리 및 분석 플랫폼으로, Hadoop 생태계를 확장.

2017년
- Kubernetes 1.0 안정화
   - 컨테이너 오케스트레이션의 표준 도구로 자리 잡음.
- React와 Vue.js 확산
   - 오픈소스 프론트엔드 프레임워크가 웹 애플리케이션 개발의 핵심 도구로 자리 잡음.
- AWS의 Kubernetes 서비스(EKS) 발표
   - 클라우드 공급자들이 Kubernetes를 통합하여 클라우드 네이티브 애플리케이션의 대중화를 가속화.

2010년대 후반 (2018~2019): 오픈소스의 주류화
2018년
- Microsoft, GitHub 인수
   - Microsoft가 GitHub를 75억 달러에 인수하며, 오픈소스 커뮤니티와의 협력을 강화.
- Kubernetes 생태계의 확장
   - Helm, Prometheus, Istio와 같은 도구들이 등장하며 클라우드 네이티브 환경이 더욱 발전.
- PyTorch 확산
   - Facebook의 딥러닝 프레임워크 PyTorch가 AI 연구와 개발의 주요 도구로 부상.

2019년
- Red Hat, IBM에 인수
   - IBM이 340억 달러에 Red Hat을 인수하며, 오픈소스 상업화의 가장 큰 사례 중 하나로 기록.
- OpenJS Foundation 설립
   - Node.js와 JavaScript 생태계를 위한 재단 설립.

2020년대 초반 (2020~2024): 확장과 통합
2020년
- 오픈소스 보안 문제 증가
   - SolarWinds 해킹 사건 등으로 오픈소스 공급망 보안에 대한 우려 증가.
   - 오픈소스 보안 자동화 도구와 정책 강화.
- Streamlit의 인기 상승
   - 데이터 애플리케이션 구축을 위한 오픈소스 프레임워크로 급성장.

2021년
- Log4j 취약점 이슈
   - 오픈소스 로그 라이브러리에서 발견된 심각한 취약점으로 전 세계적인 보안 위기가 발생.
   - 오픈소스 프로젝트의 보안 유지 필요성이 강조됨.
- Copilot 출시
   - GitHub이 AI 코드를 추천하는 도구를 공개, 개발 생산성을 혁신.

2022년
- OpenAI, ChatGPT 출시
   - 언어 모델과 AI를 활용한 툴로 오픈소스와 AI 기술의 융합 가속화.
- Rust 언어 확산
   - 메모리 안전성을 제공하는 언어로, 주요 소프트웨어 프로젝트에서 채택.

2023년
- AI 오픈소스 생태계 성장
   - Hugging Face와 같은 플랫폼을 통해 AI 모델의 오픈소스화가 가속화됨.
- Cloud Native 생태계의 성숙
   - Kubernetes와 DevOps 도구가 클라우드 환경에서 표준으로 자리 잡음.

현재(2024년)
- 오픈소스와 AI의 통합
   - AI 모델과 데이터셋의 오픈소스화로 협력과 발전이 가속화.
- 보안 중심의 오픈소스 강화
   - 공급망 보안을 위한 Sigstore, SBOM(소프트웨어 빌드 재료 명세) 도입.
- 멀티 클라우드와 엣지 컴퓨팅의 확산
   - Kubernetes 기반 오픈소스 기술이 멀티 클라우드와 엣지 컴퓨팅에서 핵심 역할.
 
<h3>2010~현재 주요 특징 요약</h3>

1. 클라우드 네이티브와 컨테이너화
- Kubernetes, Docker 등 클라우드 네이티브 기술이 오픈소스 생태계의 중심.

2. AI와 데이터 중심 기술 확산
- TensorFlow, PyTorch, Hugging Face 등 AI 프레임워크와 데이터 처리 도구가 발전.
  
3. 보안과 안정성 강화
- 공급망 보안 및 취약점 대응 기술에 대한 중요성이 증가.
  
4. 대기업의 적극적 참여
- Microsoft, Google, IBM, Amazon 등 주요 기업이 오픈소스 프로젝트에 투자.
  
5. 커뮤니티와 협업 강화
- GitHub, GitLab 등 플랫폼을 통해 글로벌 협업과 오픈소스 기여가 확대.
